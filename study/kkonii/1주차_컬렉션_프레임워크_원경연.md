## [ TOPIC ] 컬렉션 프레임워크에 대해서 설명해주세요.

---

### - 프레임워크란?
  `frame`[규칙 / 틀] + `work`[동작/ 일]  
  프로그램 개발을 위해 사용되는 API, 라이브러리 등의 `틀을 제공`하는 프로그램, 도구
  
### - 컬렉션 프레임워크란?
  `collection` [집합 / 모음] + `framework`[표준 개발 도구]   
  다수의 데이터를 모아놓은 `컬렉션` 을 처리하는 표준화된 방법을 제공하는 클래스의 집합
  
### - 장점?
1. 저장 공간이 `가변적`이다
2. 데이터의 `삽입`, `탐색`, `정렬`에 효과적이다
3. 자료구조, 알고리즘을 구현할 필요가 없다  
4. 따라서 `유지보수`가 편리하다

### - 컬렉션의 구성
1. 인터페이스 (interface)
   - 각 컬렉션에 대한 인터페이스
   - 같은 인터페이스에 대한 구현체는 `일관된 기능`을 가지고 있다
2. 클래스 (class)
   - 인터페이스를 구현한 `구현체`
   - 목적에 따라 `형태와 성능`이 다르다
3. 알고리즘 (algorithm)
   - 데이터를 다룰 때 필요한 메서드 (검색, 정렬 등)

### - 인터페이스의 종류
1️⃣ List
  - List<E>의 배열형 인터페이스
  - 데이터의 `순서가 유지`되고, `중복을 허용`

2️⃣ Set
  - Set<E> 형태의 인터페이스
  - 데이터의 `순서를 유지하지 않고`, `중복도 허용하지 않음`   
  - 

3️⃣ Map
  - <K, V>로 구성된 인터페이스
  - `[키, 값]` 을 쌍으로 매핑하여 저장하는 것이 특징
  - `해싱(hashing)` 을 사용하므로, 방대한 양의 데이터 검색에 용이

### - 클래스의 종류 1) List
1. ArrayList
   - List<E> 를 구현
   - 컬렉션 프레임워크 중 가장 많이 사용되는 클래스
   - 저장 공간을 초과하는 경우, 더 큰 배열을 생성해 기존 배열을 복사
   - 데이터의 `추가`,`삭제`를 처리하는 속도가 느리다

2. LinkedList
   - `불연속적`으로 저장된 데이터를 `연결`한 형태의 자료구조
   - 데이터의 효율적인 `추가`, `삭제`, `변경`에 용이하다
   - 저장된 요소인 `node`는 연결된 노드의 주소와 데이터값으로 구성

### - 클래스의 종류 2) Set
1. HashSet
   - Set 인터페이스를 구현한 대표적인 클래스
   - 입력 순서를 보장하지 않는다

2. LinkedHashSet
   - HashSet 과 다르게 입력 순서를 보장한다

3. TreeSet
   - 이진 검색 트리(binary search tree) 형태로 데이터를 저장하는 클래스   
  ```    
                    🟡
                   /  \
                  🔵  🟤 ---- ⭕
                 /  \   \     /  \
                🟢  ⚪  ⬛ ⬜  🔴 
  ```  
   
  - 중복 저장을 허용하지 않고, 순서도 보장하지 않는다
  - `정렬`과 `검색`에 특화되어 있다 ← [이진 탐색 트리](https://ratsgo.github.io/data%20structure&algorithm/2017/10/22/bst/)
  - 노드의 `추가`와 `삭제`에는 성능이 떨어진다

### - 클래스의 종류 3) Map
1. HashMap
   - <K, V>를 `Object`의 형태로 저장
   - Key는 `중복 불가`, value는 `중복 허용`
   - 자료형에 기본형(primitive)는 사용할 수 없다
   - 클래스, 인터페이스 타입만 가능
   - `null`을 허용한다
   - 입력 순서를 `보장하지 않는다`

2. LinkedHashMap
   - 입력된 순서를 `보장하는` 자료구조

3. TreeMap
   - TreeSet과 동일하게 이진 탐색 트리 형태로 데이터를 저장
   - key 값에 따라 오름차순으로 `자동 정렬`이 가능하다

### - 클래스의 종류 4) 공통 인터페이스
1.  Iterator
    - 컬렉션에 저장된 데이터를 읽어오는 인터페이스
    - 각 컬렉션 인터페이스에 정의된 `iterator()` 메서드는 Iterator를 반환
    - `읽어올 요소의 존재 여부 확인`, `다음 요소 읽기`, `읽어온 요소 삭제` 등의 기능

2. Comparator / Comparable
   - 컬렉션 `정렬 기준`을 정의한 인터페이스
   - 두 객체를 비교하는 메서드를 제공
     - 비교값과 같으면 `0`, 보다 크면 `양수`, 보다 작으면 `음수`
   - `Comparable`의 구현체는 기본적으로 `오름차순 정렬`을 따른다
   - `Comparator`를 구현해 다른 형태의 정렬을 정의할 수 있다
