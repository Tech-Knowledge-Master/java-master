# Try with Resource
사용 후에 반납해야하는 자원들은 Closable 인터페이스를 통해 자원을 직접 반납해야 한다.

하지만 Try-Catch-Finally를 통해 자원이 null로 변겨오디었는지 확인하고, null인 경우 자원을 반납하는 코드를 작성해야 했었다.

JVM 7 이후에 AutoCloseable 인터페이스를 구현하고 있는 자원에 대해서 try-with-resource를 적용할 수 있도록 변경되었고, 이를 사용해서 자원을 개발자가 직접 반납하는 것이 아니라 자동으로 반납하는 기능을 갖추게 되었다.

## Try-With-Resource의 장점
- 코드를 간결하게 만들 수 있음 
- 번거로운 자원 반납 작업을 하지 않아도 됨 
- 실수로 자원을 반납하지 못하는 경우를 방지할 수 있음 
- 에러로 자원을 반납하지 못하는 경우를 방지할 수 있음 
- 모든 에러에 대한 스택 트레이스를 남길 수 있음
